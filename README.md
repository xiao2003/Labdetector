NeuroLab Hub 是面向科研实验室场景构建的分布式多模态智能系统。项目以 PC 中心端与 Raspberry Pi 边缘节点协同为基础，集成实验现场监控、专家模型编排、知识库检索增强、语音交互、实验档案沉淀以及训练工作台等能力，覆盖实验室日常运行、风险识别、知识积累、模型训练与持续迭代等环节。

系统当前采用“边缘采集、中心调度、专家分析、知识增强、训练回灌”的整体路线。Pi 端负责视频采集、事件触发、语音输入与节点执行；PC 端负责多节点管理、专家调度、知识检索、结果播报、档案写入以及训练任务控制。项目适用于实验室安全巡检、实验过程记录、知识沉淀、教学演示、成果展示和 AI for Science 场景下的研究辅助。

当前交付基线在原有体系上，重点聚焦三条最小可用闭环：

* 实验室着装安全
* 危化品识别与播报
* 语音记录、整轮提取与知识入库

### 核心特性
* **解耦的 PC-Pi 双端架构**：
  项目在目录结构、运行环境和构建产物层面区分 `pc/` 与 `pi/` 两端。PC 中心端负责主控调度、专家分析与桌面交互，Pi 边缘端负责摄像头接入、边缘语音与事件上报，便于多节点扩展、单端部署、版本管理和问题隔离。
* **可编排的专家模型体系**：
  系统通过统一专家注册表、专家管理器与闭环协议组织危化品识别、仪器操作规范、PPE 穿戴、火焰烟雾、液体洒漏、手部姿态、实验问答、设备 OCR、微纳流体与纳米力学分析等专家能力，并支持综合安全聚合专家进行统一判断。
* **聚焦闭环策略**：
  当前交付版本默认只让 `PPE`、`危化品` 与 `语音记录` 进入 PC-Pi 闭环，以优先确保实验室本地闭环可用性；其他专家仍保留在系统中，但不作为默认闭环能力下发到 Pi。
* **多作用域知识库机制**：
  知识库划分为 `common` 公共底座知识库与 `expert.*` 专家专属知识库。系统支持文本、Markdown、图片、音频、视频等资料导入，并以向量索引与结构化数据库并行组织，为实验问答、专家分析、结果解释和经验回灌提供支撑。
* **统一多模态解释层**：
  前端传统 CV 专家负责检测、分类、OCR 和规则判断；在需要专业说明时，再结合对应知识库交给统一多模态 LLM 做专业研判，并将结论回传给 Pi 进行播报。
* **结构化日志、自检与实验档案**：
  PC 端提供结构化系统事件流、启动自检与实验档案中心。关键帧、事件、专家分析结果、语音轮次和会话元数据可统一归档，用于追溯、复盘、教学演示和项目汇报。
* **训练工作台与模型回灌**：
  项目内置 LLM 微调工作台与 YOLO 训练工作台，支持数据导入、训练工作区构建、训练日志记录、产物注册和运行目录回灌，形成围绕真实实验数据的持续迭代链路。
* **轻量入口交付策略**：
  当前发布采用“轻量入口 EXE + 首次自检补齐依赖”的策略，入口包保持在几十 MB 量级，重依赖和运行时在首次启动 / 自检阶段补齐，更适合实际交付和发布。

## 项目定位

NeuroLab Hub 面向以下几类典型问题：

* 实验现场监控与记录往往依赖人工值守，缺少连续、结构化、可追溯的采集与分析能力。
* 实验室风险识别通常依赖人员经验，难以形成稳定的规则、模型和闭环处理机制。
* 实验语音问答、风险提示和过程说明分散在不同工具中，缺少统一交互入口。
* 实验 SOP、设备手册、FAQ、历史记录和多媒体资料难以沉淀为可检索知识库。
* 训练数据、模型产物和运行系统之间通常割裂，缺少从数据到训练、再到部署回灌的闭环。

围绕上述问题，项目以软件系统化交付为目标，形成以“监控、分析、问答、归档、训练、回灌”为主线的实验室智能中枢方案。

## 典型应用场景

* 实验室安全巡检与异常行为识别
* 危化品容器、标签和操作规范检查
* 多实验台、多房间、多 Pi 节点集中监控
* 实验过程中的语音问答、语音记录与风险播报
* SOP、设备文档、FAQ 与经验数据的知识化管理
* 基于真实实验数据的视觉检测训练与 LLM 微调
* AI for Science 场景中的微纳流体、纳米力学过程观测与辅助分析
* 面向项目申报、验收答辩、软著材料与成果展示的系统化演示平台

## 技术路线

系统采用分层协同的方式组织运行链路：

1. **边缘采集层**：Pi 节点负责摄像头接入、边缘规则触发、语音采集和基础事件上报。
2. **中心调度层**：PC 中心端负责节点扫描、连接管理、策略下发、事件接收与状态展示。
3. **专家分析层**：专家模型基于图像、文本、事件和知识库上下文进行专业分析。
4. **知识增强层**：RAG 与结构化知识库为实验问答和专家分析提供背景信息、规则依据和历史支撑。
5. **结果反馈层**：分析结论通过桌面界面、日志系统或 Pi 端播报链路反馈给操作人员。
6. **档案沉淀层**：关键帧、事件、问答、分析结果与会话元数据统一写入实验档案。
7. **训练回灌层**：用户导入真实数据，通过训练工作台生成模型产物，并回灌到运行目录中。

## 运行闭环说明

### 1. 监控与专家分析闭环
1. PC 启动后扫描局域网内的 Pi 节点并建立连接。
2. 管理员在桌面端选择运行模式、AI 模型和相关策略。
3. PC 向指定节点下发策略与运行参数。
4. Pi 端根据摄像头输入或边缘规则触发关键帧采集和事件上报。
5. PC 接收事件后调用对应专家模型、知识库和统一多模态解释层完成分析。
6. 分析结果写入系统事件流与实验档案。
7. 结果按配置通过桌面端展示或回传 Pi 端进行文字 / 语音播报。

### 2. 语音交互闭环
1. 用户通过 PC 或 Pi 端输入语音。
2. 本地语音状态机先区分：唤醒、控制指令、记录类、专家唤醒和普通问答。
3. 普通问答结合知识库与模型后端生成回答。
4. 危化品等专家型语音命令会直接进入对应专家链路。
5. 整轮语音结束后，仅从用户口述中提取有效知识并写入知识库。

### 3. 知识增强闭环
1. 管理员导入公共知识和专家专属知识资料。
2. 系统对资料进行整理、切分、索引与结构化存储。
3. 专家分析和问答过程调用对应知识作用域。
4. 新产生的记录、总结和多媒体摘要可继续回灌至知识库。

### 4. 训练回灌闭环
1. 用户导入实验数据、语料或标注结果。
2. 训练工作台构建训练工作区并检查运行环境。
3. 用户执行 LLM 微调或视觉模型训练。
4. 系统输出训练日志、模型产物和注册信息。
5. 训练结果可回灌至运行目录并重新投入使用。

## 功能模块说明

### 1. 桌面中心端
桌面中心端主要由 `pc/desktop_app.py` 驱动，当前包括以下功能：

* 会话状态展示、顶部操作入口与系统概览
* 单机监控与多节点监控墙
* 运行配置、模型选择与运行模式切换
* 知识库管理、专家模型管理、模型配置
* 启动自检、软件说明、版权页和相关子窗口
* 训练工作台与实验档案中心入口

### 2. 多节点扫描与通信模块
相关模块主要位于 `pc/communication/` 与 `pi/pisend_receive.py`，支持以下能力：

* 局域网节点扫描
* WebSocket 多节点管理
* 中心端策略下发
* 关键帧与事件回传
* 专家分析结果回传 Pi 端
* 节点状态同步、结果闭环与运行审计

### 3. 启动自检与运行检查
PC 与 Pi 两端均提供自检与环境检查能力：

* PC 端在 `pc/webui/runtime.py` 中完成运行环境、模型服务、知识库目录和训练依赖检查。
* Pi 端通过 `pi/pi_cli.py` 和相关运行流程完成命令、依赖和节点环境检查。
* 发布流程中要求执行编译检查、功能测试、入口烟测和文档一致性检查。

### 4. 实验档案中心
实验档案能力位于 `pc/core/experiment_archive.py`，用于沉淀：

* 会话元数据
* 关键帧文件
* 事件类型与事件内容
* 风险提示与专家分析结果
* 语音交互内容
* 时间线记录与会话关闭信息

该模块用于支撑实验过程追溯、示范材料制作、实验安全复盘和知识抽取。

### 5. 知识库系统
知识库能力位于 `pc/knowledge_base/`，包括：

* `rag_engine.py`：向量检索与知识管理主入口
* `structured_kb.py`：结构化 SQLite 知识库
* `media_ingestion.py`：图片、音频、视频资料导入与摘要处理

当前知识库支持：

* `common` 公共底座知识库
* `expert.*` 专家专属知识库
* 文本、Markdown、图片、音频、视频导入
* 结构化记录与向量索引并行组织

### 6. 专家模型体系
专家注册表位于 `pc/core/expert_registry.py`，专家加载与调度由 `pc/core/expert_manager.py` 负责。当前已接入的核心专家包括：

#### 实验室安全规范相关专家
* `safety.chem_safety_expert`
* `safety.equipment_operation_expert`
* `safety.flame_fire_expert`
* `safety.general_safety_expert`
* `safety.hand_pose_expert`
* `safety.integrated_lab_safety_expert`
* `safety.ppe_expert`
* `safety.spill_detection_expert`

#### 实验知识与问答相关专家
* `lab_qa_expert`
* `equipment_ocr_expert`

#### AI for Science 研究辅助相关专家
* `nanofluidics.microfluidic_contact_angle_expert`
* `nanofluidics.nanofluidics_multimodel_expert`

### 7. AI 模型与模型配置
当前 GUI 运行配置中，`AI模型` 采用“本地优先、云端补充”的策略：

* `Ollama`
* `通义千问（需在模型配置页面添加 API Key）`
* `DeepSeek（需在模型配置页面添加 API Key）`
* `Kimi（需在模型配置页面添加 API Key）`
* `本地模型框架`

其中：

* `Ollama` 置顶，作为默认本地闭环方案
* 云端模型统一通过 GUI 的“模型配置”录入，不预塞默认模型
* 保存配置后，会把模型名称写回左侧“模型选择”下拉框

### 8. 训练工作台
训练相关模块位于 `pc/training/`，当前支持：

* 导入 LLM 微调语料
* 导入 YOLO 检测训练数据
* 构建训练工作区
* 检查训练依赖与本地运行时
* 启动训练任务
* 记录训练日志
* 自动注册训练结果并回灌运行目录

## 硬件准备与部署清单

为保障多节点视频传输、专家分析和训练任务的稳定运行，建议参考以下硬件配置：

### 1. 中心计算节点 (PC 端)
负责多节点管理、专家分析、知识检索、语音理解、桌面交互与训练控制。
* **操作系统**：Windows 10 / Windows 11，64 位。
* **算力设备 (GPU)**：建议配置 NVIDIA RTX 3090 / 4090 / 5090 或同等级设备，用于视觉推理与训练任务。
* **内存 (RAM)**：建议 32GB 及以上，以保障多节点视频帧缓存、知识库索引与训练工作区稳定运行。
* **存储空间**：建议预留 50GB 以上空间，用于模型、知识库、训练产物和实验档案存储。
* **外设交互**：如需本机语音交互，建议配置麦克风与音频输出设备。

### 2. 边缘节点 (Raspberry Pi)
负责实验位点的视频采集、事件触发、语音输入与边缘执行。
* **主控板**：Raspberry Pi 4B / 5 或同等级 ARM Linux 设备。
* **摄像头**：CSI 或 USB 摄像头，需与当前采集链路兼容。
* **音频设备**：根据场景配置麦克风、扬声器或语音模块。
* **存储与供电**：建议 32GB 以上存储，并使用稳定电源保障长时运行。

### 3. 网络环境要求
* **网络拓扑**：PC 与全部 Pi 节点需处于同一局域网。
* **链路稳定性**：建议使用稳定的千兆网络或高质量 Wi-Fi 环境，以支撑多节点并发视频与事件回传。
* **模型服务连通性**：如使用本地或云端模型服务，需保证中心端与对应服务地址连通。

## 上手指南

### 0. 获取源码

```bash
git clone https://github.com/xiao2003/Labdetector.git
cd Labdetector
```

### 1. 中心端部署 (PC)
在项目根目录执行环境安装：

```bash
python -m venv .venv
.venv\Scripts\activate
pip install -r requirements.txt
```

如需源码方式启动主程序，可执行：

```bash
python launcher.py
```

如需直接使用打包产物，可使用轻量入口与发布包。

### 2. 边缘端部署 (Raspberry Pi)
将 `pi/` 目录部署到边缘设备后，按需安装依赖并启动：

```bash
cd pi
./start_pi_node.sh start --auto-install-deps
```

如仅需查看命令帮助，可执行：

```bash
python pi_cli.py --help
```

### 3. 知识库与专家能力准备
建议在正式运行前完成以下准备：

1. 导入 `common` 公共底座知识库资料。
2. 按需导入对应 `expert.*` 专家知识库资料。
3. 为需要模型文件的专家导入模型资产或样本文件。
4. 在桌面端完成知识库、专家和模型配置检查。

### 4. 训练工作台
系统提供两个训练入口：

* `launcher.py --annotation-panel`：标注与视觉训练入口
* GUI 中的训练中心：LLM / YOLO 训练管理入口

当前支持：

* LLM 微调语料导入
* 视觉检测数据导入
* 训练工作区构建
* 训练依赖检查
* 训练结果注册与回灌

### 5. 自动化测试与发布验收
当前版本 `V3.0.9` 已完成：
