## 使用规则
- 只允许修改 `D:\Labdetector`
- 不允许修改工作区外目录
- 不允许执行破坏性命令
- 每次只完成一个完整模块
- 每完成一个模块后必须：
  - 更新相关文档
  - 执行自动化测试环节
  - 进行最小验证
  - 清理临时文件
  - 输出结果摘要
- 如遇必须人工决策的问题，再暂停

## 模块完成后的自动化测试环节
每完成一个任务模块后，必须按顺序执行以下检查：

1. 运行与本模块直接相关的 Python 编译检查
   - 示例：`python -m py_compile <相关文件>`
2. 运行与本模块直接相关的最小功能测试
   - 界面模块：至少验证入口可启动、核心按钮路径可打开
   - 训练模块：至少验证配置页、任务创建、结果输出路径
   - 数据模块：至少验证导入、检索、导出一条完整路径
3. 如果模块涉及桌面入口、训练入口或 Pi CLI，则补充运行级烟测
   - PC 主程序：`pc/NeuroLab Hub.exe --smoke-test-file <输出文件>`
   - LLM 入口：`pc/NeuroLab Hub LLM.exe --smoke-test-file <输出文件>`
   - Vision 入口：`pc/NeuroLab Hub Vision.exe --smoke-test-file <输出文件>`
   - Pi CLI：`python pi/pi_cli.py --help` 或对应子命令 `--help`
4. 如果模块涉及文档或发版链路，则补充一致性检查
   - 检查 README、用户手册、软件说明书是否同步更新
   - 检查正式名称、简称、版本号是否一致
5. 输出固定格式的测试摘要
   - 测试对象
   - 执行命令
   - 结果
   - 未解决风险

自动化执行要求：
- 优先使用可脚本化命令，不依赖人工点击
- 如果某项测试因环境缺失无法执行，必须明确记录原因
- 不允许跳过测试直接标记任务完成
