# NeuroLab Hub 软件著作权申请材料整理稿

版本：3.0.6  
文档用途：用于准备计算机软件著作权登记时的软件说明、功能摘要、材料清单和提交组织方式

## 一、使用说明

本文件用于将当前仓库和已交付软件整理为一套适合软著申报的材料。文中给出的软件名称、功能简介、技术特点、运行环境、模块说明和材料清单，可直接作为申报表、说明书和附件材料的编写基础。

本文件不是法律意见书。具体提交口径、页数和格式，应以提交当日中国版权保护中心登记系统和受理要求为准。

## 二、主要依据

### 2.1 法规依据

1. 《计算机软件保护条例》  
   来源：[中国网信网](https://www.cac.gov.cn/2013-02/20/c_126511511.htm)

### 2.2 材料组织说明

根据《计算机软件著作权登记办法》，软件著作权登记通常围绕以下几类材料组织：

- 软件基本信息
- 申请人主体信息
- 软件鉴别材料
- 权利归属补充材料（如合作开发、委托开发、受让、继承、修改他人软件等情形）

对当前项目而言，如软件由当前团队独立开发，一般重点准备以下内容：

1. 申请表基础信息
2. 申请人身份证明或主体证明
3. 软件源程序节选
4. 软件文档节选

## 四、可直接复用的软件简介

NeuroLab Hub 是一套面向科研实验室场景的多模态智能软件系统。软件以 AI for Science 为技术导向，融合计算机视觉、大语言模型、语音识别、语音播报、知识库管理、实验档案沉淀和模型训练能力，面向实验室监控、风险预警、智能咨询、知识传承和模型持续训练等需求，提供一体化的软件化解决方案。系统采用 PC 中心端与 Raspberry Pi 边缘节点协同架构，支持多节点视频接入、关键帧分析、专家模型调度、知识增强问答、语音播报和模型训练回灌，可直接以安装包、便携版和源码仓库形式交付。

## 五、可直接复用的开发目的说明

本软件旨在解决实验室管理与科研辅助场景中存在的实时监控不足、操作指导效率低、实验知识沉淀困难、风险提示滞后以及用户私有模型训练闭环缺失等问题。通过构建集视觉感知、语义理解、语音交互、知识增强、实验档案和模型训练为一体的软件平台，提高实验室智能化管理水平，增强实验过程的可视化、可追溯性和知识复用能力。

## 六、可直接复用的主要功能说明

7. LLM 微调与识别模型训练工作台
8. 模型结果回灌与持续迭代

### 6.2 主要模块说明

- 桌面可视化监控模块
- Pi 节点扫描与调度模块
- 视觉识别与专家分析模块
- 语音智能交互模块
- 知识库管理模块

## 七、可直接复用的技术特点说明

1. 采用 PC 中心端与 Raspberry Pi 边缘节点协同架构，支持多节点实验场景接入。
2. 采用专家模型体系组织实验室安全、仪器操作、OCR、实验问答和 AI for Science 研究分析能力。
3. 采用 `common + expert.*` 的多知识库结构，支持公共底座知识与专家专属知识并行增强。
4. 支持本地和云端大模型服务接入，兼容 Ollama、LM Studio、vLLM、SGLang、LMDeploy、Xinference、llama.cpp 和 OpenAI 兼容 API。
5. 支持将实验运行、语音交互和关键事件结构化归档。
6. 支持基于真实实验数据进行 LLM 微调和识别模型训练，并将训练产物回灌到运行环境。
7. 以安装包、便携包和完整包三种形式交付，降低普通用户使用门槛。

## 八、可直接复用的运行环境说明

### 8.1 PC 端

- 操作系统：Windows 10 / Windows 11 64 位
- CPU：8 核以上推荐
- 内存：32GB 及以上推荐
- GPU：建议具备 NVIDIA RTX 系列独立显卡
- 网络：可与 Pi 节点通信
- 存储：建议预留 50GB 以上空间

### 8.2 Pi 端

- 设备：Raspberry Pi 4B / 5 或同等级 ARM Linux 设备
- 外设：摄像头、麦克风、扬声器
- 网络：与 PC 中心端可达

### 8.3 软件环境

- Python 3.11 生态运行时
- OpenCV、Tk、语音处理与 WebSocket 相关依赖
- 可选本地模型服务或云端 API

## 十、建议的材料组织方式

## 十三、申报时容易遗漏的问题

1. 软件全称、简称和版本号在各材料中不一致。
2. 软件说明书写成仓库说明，而不是软件产品说明。
3. 源程序节选未覆盖主流程和核心模块。
4. 材料中仍保留旧项目名或旧版本号。
5. 申请表中的功能摘要过短，无法体现软件完整性。
6. 运行环境、操作对象和适用场景描述过于笼统。
7. 测试材料与实际交付版本不一致。
