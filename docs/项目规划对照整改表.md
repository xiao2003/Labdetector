# 项目规划对照整改表

| 模块 | 当前状态 | 优先级 | 对应代码位置 | 整改说明 |
| --- | --- | --- | --- | --- |
| PC-Pi 视觉闭环 | 已实现 | 高 | `pc/webui/runtime.py`、`pc/communication/`、`pi/pisend_receive.py` | 支持节点扫描、策略下发、关键帧回传、专家分析、结果回传播报。 |
| 多节点视频可视化 | 已实现 | 高 | `pc/desktop_app.py` | 右侧监控墙支持多 Pi 视频平铺与点击放大查看。 |
| 语音交互闭环 | 已实现 | 高 | `pc/voice/voice_interaction.py`、`pi/voice/` | 支持 Pi 端远程语音交互与 PC 本机语音交互。 |
| 语音轮次归档 | 已实现 | 高 | `pc/core/voice_round_archive.py` | 单轮运行按对话轮次归档，输出 JSON 与 Markdown。 |
| 专家模型导入 | 已实现 | 高 | `pc/core/expert_manager.py`、`pc/webui/server.py` | 支持界面与 HTTP 接口导入专家资产。 |
| 多知识库架构 | 已实现 | 高 | `pc/knowledge_base/rag_engine.py` | 支持 `common` 与 `expert.*` 多作用域知识库。 |
| 媒体资料导入 | 已实现 | 高 | `pc/knowledge_base/media_ingestion.py`、`pc/knowledge_base/media_semantics.py` | 图片、音频、视频导入时生成语义摘要与 sidecar 元数据。 |
| 风险规则外置 | 已实现 | 中 | `pc/experts/safety/risk_rulebook.py`、`pc/experts/safety/risk_rules.json` | 风险等级阈值和规则已从代码移出，支持后续扩展。 |
| 实验档案中心 | 已实现 | 高 | `pc/core/experiment_archive.py` | 按项目、实验、人员、时间归档运行事件。 |
| 训练工作台 | 已实现 | 高 | `pc/training/`、`pc/desktop_app.py` | 支持构建训练工作区、导入真实数据、启动微调任务。 |
| LLM 微调 | 已实现 | 高 | `pc/training/llm_finetune.py` | 支持基于 JSONL 数据的 LoRA/SFT 微调。 |
| Pi 检测模型微调 | 已实现 | 高 | `pc/training/pi_detector_finetune.py` | 支持基于 YOLO 数据集的检测模型训练与部署。 |
| 一键训练流程 | 已实现 | 高 | `pc/training/train_manager.py`、`pc/webui/runtime.py` | 支持真实数据导入、工作区构建、LLM 与 Pi 模型顺序训练。 |
| GitHub 面向用户发布 | 已实现 | 高 | `.github/workflows/`、`installer/`、`scripts/` | 普通用户通过 GitHub Release 下载安装器，开发者使用源码仓库。 |
| 视觉语义统一 schema | 待增强 | 中 | `pc/experts/safety/semantic_risk_mapper.py` | 当前已提供结构化语义字段，后续可补更严格的实验流程语义表。 |
| 行为识别与步骤时序模型 | 待增强 | 中 | `pc/training/`、`pi/edge_vision/` | 当前以检测与专家规则为主，后续可引入时序动作识别。 |
| AI for Science 深层科研辅助 | 待增强 | 中 | `pc/experts/` | 后续可扩展实验机理分析、实验结果解释与研究决策辅助专家。 |

## 整改原则

1. 先保证“真实可运行”的监控、语音、归档、训练闭环。
2. 再补“真实数据驱动”的标注、微调、评估与持续迭代流程。
3. 最后扩展更强的 AI for Science 研究型能力，而不是只停留在软件外壳层面。
