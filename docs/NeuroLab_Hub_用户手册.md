# NeuroLab Hub——可编排专家模型的实验室多模态智能中枢 用户手册

版本：1.0.0  
适用对象：实验室管理人员、实验操作人员、部署维护人员、模型维护人员

## 1. 手册说明

本手册用于说明 NeuroLab Hub 正式版 1.0.0 的获取、启动、自检、主界面、知识库、专家模型、Pi 节点接入、语音交互、训练工作台与常见问题。本文档与当前 GitHub 仓库、交付包和正式版 Release 保持一致。

## 2. 软件简介

NeuroLab Hub 是一套面向实验室场景的中心-分布式多模态智能系统。系统由 PC 中心端、Pi 边缘节点、专家模型体系、多知识库体系和训练工作台构成，适用于实验室安全监测、危化品识别、知识沉淀、语音交互和模型持续迭代。

## 3. 软件组成

### 3.1 Windows 端

- `pc/NeuroLab Hub.exe`：主程序入口
- `pc/NeuroLab Hub LLM.exe`：LLM 工作台入口
- `pc/NeuroLab Hub Vision.exe`：视觉训练入口

### 3.2 Pi 端

- `pi/start_pi_node.sh`：一键启动脚本
- `pi/pi_cli.py`：Pi 端命令行入口
- `pi/pisend_receive.py`：Pi 端主控制流

## 4. 获取方式

### 4.1 用户交付包

普通用户下载 `NeuroLab-Hub-v1.0.0.zip`，解压后进入 `pc/` 目录运行主程序即可。

### 4.2 源码包

开发者下载 `NeuroLab-Hub-source-v1.0.0.zip`，解压后即可按 README、测试手册和源码目录继续开发与联调。

## 5. 安装与启动

### 5.1 Windows 端启动

1. 解压 `NeuroLab-Hub-v1.0.0.zip`
2. 打开 `pc/` 目录
3. 双击 `NeuroLab Hub.exe`
4. 进入启动引导与系统自检
5. 等待程序自动补齐缺失依赖和模型资产
6. 自检完成后进入主界面

### 5.2 Pi 端启动

推荐先执行自检，再启动节点服务：

```bash
./start_pi_node.sh self-check --auto-install-deps
./start_pi_node.sh start --auto-install-deps
```

## 6. 首次使用流程

1. 启动 PC 主程序并完成系统自检
2. 选择 `AI模型` 和 `模型选择`
3. 如需云端模型，在 `模型配置` 中添加 API Key 和模型名称
4. 导入 `common` 公共知识库和专家知识库资料
5. 如需 Pi 节点，先确认网络可达与设备在线
6. 启动监控并观察系统事件流

## 8. AI 模型使用说明

### 8.1 本地模型

`Ollama` 默认提供以下本地模型候选：

- `gemma3:4b`
- `llama3.2-vision:11b`
- `qwen3.5:4b`
- `qwen3.5:9b`
- `qwen3.5:27b`
- `qwen3.5:35b`

首次选择模型时，系统会自动拉取缺失模型。

### 8.2 云端模型

当前支持通过 `模型配置` 添加：

- 通义千问
- DeepSeek
- Kimi

保存配置后，模型名称会自动加入左侧对应 `AI模型` 的 `模型选择`。

## 9. 知识库管理

### 9.1 知识库结构

- `common`：公共知识库
- `expert.*`：专家专属知识库

### 9.2 导入方式

可通过知识中心导入：

- 文本文档
- Markdown
- 图片
- 音频
- 视频

## 10. 专家模型与闭环

当前正式聚焦闭环能力包括：

- 实验室着装安全
- 危化品识别与播报
- 语音记录与知识提取

系统采用“Pi 边缘触发 -> PC 专家分析 -> 知识库增强 -> 统一多模态解释 -> 回传 Pi 播报”的链路。

## 11. 语音交互

### 11.1 PC 端

- 支持唤醒词
- 支持控制指令
- 支持问答
- 支持专家唤醒
- 支持整轮记录与知识提取

### 11.2 Pi 端

- Pi 上传语音到 PC
- PC 完成处理和分析
- 结果回传 Pi 播报

## 12. 训练工作台

- `NeuroLab Hub LLM.exe`：用于 LLM 数据整理与工作台操作
- `NeuroLab Hub Vision.exe`：用于视觉训练与数据工作台操作

## 13. 常见问题

### 13.1 启动较慢

首次启动会进行依赖检测、语音模型检查和运行环境准备，属于正常现象。

### 13.2 本地模型不可用

请检查：

- Ollama 是否已安装
- 是否已联网拉取模型
- 当前模型是否已在 `模型选择` 中生效

### 13.3 Pi 节点无法接入

请检查：

- PC 与 Pi 是否在同一网络
- Pi 是否已执行自检和启动
- 防火墙是否阻断端口
