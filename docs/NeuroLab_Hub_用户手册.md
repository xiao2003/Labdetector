# NeuroLab Hub：可编排专家模型的实验室多模态智能中枢 用户手册

版本：3.0.6  
适用对象：实验室管理人员、实验操作人员、部署维护人员、模型维护人员

## 1. 手册说明

本手册用于说明 NeuroLab Hub 这一“可编排专家模型的实验室多模态智能中枢”的安装、启动、主界面组成、知识库管理、专家模型管理、Pi 节点接入、语音交互、训练工作台、日志查看和常见问题。建议初次使用者先阅读本手册，再结合主界面上的软件说明和版权页面进行操作。

## 2. 软件简介

NeuroLab Hub 是一套可编排专家模型的实验室多模态智能中枢可视化软件平台。系统由 PC 中心端、Pi 边缘节点、专家模型体系、多知识库体系和训练工作台构成，适用于实验室监控、风险提示、智能咨询、知识积累、演示展示和模型迭代等场景。

## 3. 软件组成

### 3.1 Windows 端

- `pc/NeuroLab Hub.exe`：主程序入口
- `pc/NeuroLab Hub LLM.exe`：LLM 微调工作台入口
- `pc/NeuroLab Hub Vision.exe`：识别模型训练入口

### 3.2 Pi 端

- `pi/start_pi_node.sh`：一键启动脚本
- `pi/pi_cli.py`：Pi 端命令行入口
- `pi/pisend_receive.py`：Pi 端主控制流

## 4. 获取方式

### 4.1 安装版

适合普通 Windows 用户。双击 `NeuroLab-Hub-Setup-vX.Y.Z.exe` 后按引导安装即可。

### 4.2 便携版

适合无需安装的快速使用场景。解压 `NeuroLab-Hub-Portable-vX.Y.Z.zip` 后直接进入 `pc/` 目录运行。

### 4.3 完整包

适合同时部署 PC 和 Pi 的完整闭环。解压 `NeuroLab-Hub-vX.Y.Z.zip` 后，将 `pc/` 目录用于中心端，将 `pi/` 目录复制到边缘节点。

## 5. 安装与启动

### 5.1 Windows 安装版

1. 双击 `NeuroLab-Hub-Setup-vX.Y.Z.exe`。
2. 阅读安装信息与许可协议。
3. 选择安装目录。
4. 等待安装完成。
5. 从桌面或开始菜单启动 `NeuroLab Hub`。

### 5.2 Windows 便携版

1. 解压 `NeuroLab-Hub-Portable-vX.Y.Z.zip`。
2. 打开 `pc/` 目录。
3. 双击 `NeuroLab Hub.exe` 进入主程序。
4. 如需 LLM 微调，双击 `NeuroLab Hub LLM.exe`。
5. 如需识别模型训练，双击 `NeuroLab Hub Vision.exe`。

### 5.3 Pi 端启动

推荐先执行自检，再启动节点服务：

```bash
./start_pi_node.sh self-check --auto-install-deps
./start_pi_node.sh start --auto-install-deps
```

## 6. 首次使用建议

1. 启动 PC 主程序后先执行系统自检。
2. 先配置 AI 后端和模型服务，再接入业务功能。
3. 先导入 `common` 公共底座知识库，再按专家导入专属知识库。
4. 再根据实验场景启用对应专家模型。
5. 如使用 Pi 节点，先验证局域网可达和摄像头可用。
6. 正式运行前，建议先用演示数据或测试节点验证闭环。

## 7. 主界面说明

### 7.1 顶部品牌区

- 显示软件名称 `NeuroLab Hub`
- 显示版本号和运行状态
- 右上角核心按钮横向排列，可进入自检、知识库、专家模型、模型服务、训练工作台、软件说明和版权页面

### 7.2 左侧配置区

- AI 后端选择
- 模型预设选择
- 自定义模型输入
- 运行模式选择
- 预期节点数设置
- 启动监控与停止监控
- 系统概览与启动自检结果展示

### 7.3 右侧主工作区

- 上半部分：多节点监控墙，平铺展示多个 Pi 节点视频
- 中间区域：节点状态卡片和核心提示信息
- 下半部分：系统事件流，以结构化表格展示时间、级别、模块和摘要
- 底部区域：当前选中事件的详细内容，用于查看完整告警或错误说明

## 8. 系统事件流说明

系统事件流用于替代早期跳屏刷新的日志面板，适合长时间观察与正式展示：

- `时间`：记录事件时间
- `级别`：区分 INFO、WARN、ERROR 等等级
- `模块`：标识事件来源，例如运行时、知识库、Pi 节点、语音交互
- `摘要`：显示核心提示内容
- `详情`：在下方详情区查看完整信息

建议在演示、汇报和问题定位时以系统事件流为主，而不是依赖终端输出。

## 9. AI 后端与模型服务配置

### 9.1 支持方式

当前可接入：

- Ollama
- OpenAI 兼容服务
- LM Studio
- vLLM
- SGLang
- LMDeploy
- Xinference
- llama.cpp
- OpenAI、DeepSeek 等云端 API

### 9.2 配置建议

1. 若本地算力充足，优先使用本地模型服务。
2. 若使用云端 API，请妥善保管 API Key。
3. 需要视觉能力时，优先选择支持多模态输入的服务端。
4. 对实验数据安全要求较高时，建议使用本地部署模式。

## 10. 知识库管理

### 10.1 知识库结构

- `common`：全局公共知识库
- `expert.*`：各专家专属知识库

### 10.2 可导入资料

- 文本文档
- Markdown 文档
- 图片资料
- 音频资料
- 视频资料

### 10.3 推荐导入顺序

1. 导入实验室规章、SOP 和通用 FAQ 至 `common`
2. 按专家模型分别导入危化品、仪器、设备说明等专属资料
3. 导入历史实验记录和典型案例
4. 持续回灌新的实验经验和训练总结

## 11. 专家模型管理

### 11.1 当前核心专家

- 危化品识别与提醒
- 仪器操作规范检查
- PPE 穿戴检查
- 火焰烟雾风险提示
- 液体洒漏检测
- 手部姿态分析
- 设备 OCR 识别
- 实验室问答
- 微纳流体接触角分析
- 微纳力学多模型分析

### 11.2 使用建议

1. 仅启用当前实验场景所需专家，降低无关提示干扰。
2. 为各专家建立对应知识库，提高分析一致性。
3. 在正式环境中尽量避免临时替换专家配置。

## 12. Pi 节点部署与接入

### 12.1 推荐步骤

1. 将 `pi/` 目录复制到 Raspberry Pi。
2. 接入摄像头、麦克风与扬声器。
3. 执行自检并自动安装依赖。
4. 启动 Pi 节点服务。
5. 在 PC 主程序中扫描并接入节点。

### 12.2 典型闭环

1. PC 扫描 Pi。
2. PC 下发策略。
3. Pi 截取关键帧。
4. PC 接收并分析关键帧。
5. PC 输出结果。
6. Pi 接收结果并进行语音或文字播报。

## 13. 语音交互使用说明

### 13.1 Pi 端语音交互

- 适用于实验现场的边缘节点交互。
- 用户对 Pi 端麦克风讲话，问题会被上传到中心端分析并回传答案。

### 13.2 PC 端语音交互

- 适用于本机摄像头模式或演示模式。
- 可直接在 PC 上完成语音采集、理解和播报。

### 13.3 日志归档

- 单轮语音交互会按轮次归档。
- 归档内容可用于实验复盘、知识沉淀和问题追踪。

## 14. 训练工作台

### 14.1 LLM 微调入口

使用 `NeuroLab Hub LLM.exe` 进入。适合：

- 导入实验语料
- 构建微调数据集
- 检查训练依赖
- 执行微调
- 注册和回灌新模型

### 14.2 识别模型训练入口

使用 `NeuroLab Hub Vision.exe` 进入。适合：

- 导入实验图像和视频数据
- 构建识别训练数据集
- 训练检测模型
- 导出和注册训练结果

### 14.3 训练注意事项

1. 训练前确认磁盘空间与 GPU 资源。
2. 训练数据建议按项目、场景和标签规范组织。
3. 训练产物回灌前建议先在测试环境验证。

## 15. 实验档案与结果沉淀

系统会记录：

- 关键帧
- 事件日志
- 风险提示
- 对话轮次
- 训练记录
- 回灌模型信息

建议按实验项目、时间和人员维度定期整理和备份。

## 16. 常见问题

### 16.1 主程序启动慢

首次启动时可能执行依赖检测和环境准备。若长期停留，请查看系统事件流中的 ERROR 信息。

### 16.2 Pi 无法被扫描到

请检查：

- PC 与 Pi 是否在同一网络
- Pi 是否已启动服务
- 防火墙是否拦截通信端口

### 16.3 模型服务不可用

请检查：

- 本地模型服务是否已启动
- API 地址是否正确
- API Key 是否有效
- 当前模型是否支持目标任务

### 16.4 训练失败

请检查：

- 训练依赖是否已安装
- 训练数据路径是否正确
- GPU 驱动与显存是否满足要求
- 训练日志中的首个报错位置

## 18. 结语

NeuroLab Hub 的使用重点不在于单一模型能力，而在于多节点感知、知识增强、专家分析、语音交互、实验留痕和训练回灌的整体闭环。建议用户先跑通一条最小闭环，再逐步扩展到多节点、多专家和持续训练场景。
