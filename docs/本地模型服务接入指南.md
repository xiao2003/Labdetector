# 本地模型服务接入指南

本文说明如何将 LabDetector 接到本地或局域网内的大模型服务。LabDetector 自身不重复实现推理引擎，而是复用现成的 OpenAI 兼容服务接口，因此可以直接接入成熟开源框架，而不是在项目里重复造轮子。

当前桌面端已内置以下模型服务预设：

- LM Studio
- vLLM
- SGLang
- LMDeploy
- Xinference
- llama.cpp Server
- 自定义 OpenAI 兼容服务

## 1. 设计原则

LabDetector 只负责业务编排、专家路由、知识库和监控闭环；大模型推理由外部服务承载。这样做有三个直接收益：

- 用户可以自由切换 Qwen、DeepSeek 等模型，不被单一推理后端绑定
- PC 面板里只需要配置 `Base URL`、`模型名` 和可选 `API Key`
- 同一套 UI 可以同时适配云端 API 与本地私有化服务

## 2. 截至 2026 年 3 月 8 日的模型建议

我核对了官方仓库与官方文档后，当前更稳妥的建议是：

- 文本问答、知识库总结、语音交互优先使用 `Qwen3` 或 `DeepSeek` 系列
- 视觉理解、关键帧分析、多模态专家链路优先使用 `Qwen2.5-VL` 一类视觉模型
- 当前没有查到官方公开开源、且命名明确为 `Qwen3.5` 的正式仓库发布，因此软件预设仍以 `Qwen3` 文本方向和 `Qwen2.5-VL` 视觉方向组织

## 3. 软件内如何配置

在 PC 端打开“模型服务配置”，选择对应服务项后填写：

- `Base URL`
- `模型名`
- `API Key`，本地服务通常可留空

常见默认地址如下：

- LM Studio：`http://127.0.0.1:1234/v1`
- vLLM：`http://127.0.0.1:8000/v1`
- SGLang：`http://127.0.0.1:30000/v1`
- LMDeploy：`http://127.0.0.1:23333/v1`
- Xinference：`http://127.0.0.1:9997/v1`
- llama.cpp Server：`http://127.0.0.1:8080/v1`

只要服务对外暴露的是 OpenAI 兼容接口，LabDetector 就可以复用。

## 4. 推荐使用的开源项目

以下项目均来自官方仓库或官方文档，适合作为 LabDetector 的本地模型服务底座：

- [LM Studio](https://lmstudio.ai/docs/app/api/endpoints/openai)：Windows 下最快落地，适合单机演示和快速验证
- [vLLM](https://docs.vllm.ai/en/latest/serving/openai_compatible_server.html)：适合工作站或服务器 GPU 推理，提供 OpenAI 兼容服务
- [SGLang](https://docs.sglang.ai/basic_usage/openai_api_completions.html)：适合高吞吐、多模态服务，OpenAI 兼容接口成熟
- [LMDeploy](https://lmdeploy.readthedocs.io/en/latest/llm/api_server.html)：对 Qwen、DeepSeek 等中文生态模型友好，适合国产模型本地部署
- [Xinference](https://inference.readthedocs.io/en/latest/getting_started/using_xinference.html)：适合做统一模型管理和多模型服务编排
- [llama.cpp](https://github.com/ggml-org/llama.cpp)：适合 GGUF 量化模型、CPU 或轻量 GPU 环境

## 5. 典型启动命令

以下命令保持尽量贴近官方用法，便于用户直接对照部署。

### 5.1 vLLM

Qwen 文本模型示例：

```bash
vllm serve Qwen/Qwen3-8B --host 0.0.0.0 --port 8000
```

DeepSeek 文本模型示例：

```bash
vllm serve deepseek-ai/DeepSeek-R1-Distill-Qwen-7B --host 0.0.0.0 --port 8000
```

### 5.2 SGLang

Qwen 视觉模型示例：

```bash
python -m sglang.launch_server --model-path Qwen/Qwen2.5-VL-7B-Instruct --host 0.0.0.0 --port 30000
```

### 5.3 LMDeploy

Qwen 视觉模型示例：

```bash
lmdeploy serve api_server Qwen/Qwen2.5-VL-7B-Instruct --server-port 23333
```

### 5.4 llama.cpp Server

GGUF 量化模型示例：

```bash
llama-server -m /path/to/model.gguf --host 0.0.0.0 --port 8080
```

### 5.5 LM Studio

LM Studio 更适合图形界面操作：下载模型后，在软件内启用本地 API 服务，再将 `http://127.0.0.1:1234/v1` 填入 LabDetector 即可。

### 5.6 Xinference

Xinference 更适合作为统一模型管理器。先启动本地服务，再在其界面或 CLI 中拉起模型实例，最后将 `http://127.0.0.1:9997/v1` 填入 LabDetector。

## 6. 推荐的模型组合

推荐按任务类型拆分，而不是强行让一个模型包办全部场景：

- 语音问答、知识库增强、实验档案总结：`Qwen3` 或 `DeepSeek` 文本模型
- 关键帧分析、视觉专家解释：`Qwen2.5-VL` 等视觉模型
- 低资源设备快速验证：GGUF 量化模型 + `llama.cpp`
- Windows 单机演示：LM Studio
- GPU 工作站多用户接入：vLLM、SGLang、LMDeploy

## 7. 与训练工作台的关系

- `LabTraining.exe` 用于本地 LoRA 微调与 Pi 端 YOLO 检测模型训练
- “模型服务配置”用于推理阶段接入本地或云端服务
- 两者可以并行使用，例如：
  - 训练工作台产出本地微调适配器
  - 文字问答走 vLLM / LMDeploy / DeepSeek API
  - 多模态理解走 Qwen2.5-VL 本地服务

## 8. 已验证的软件接入方式

当前代码已经按通用 OpenAI 兼容接口实现接入，桌面端已预置：

- 通义千问云 API
- OpenAI 云 API
- DeepSeek 云 API
- Kimi 云 API
- 自定义 OpenAI 兼容服务
- LM Studio / vLLM / SGLang / LMDeploy / Xinference / llama.cpp 本地服务

如果某个服务支持 `/v1/models`，软件会优先自动探测模型列表；否则按用户填写的模型名直接调用。
