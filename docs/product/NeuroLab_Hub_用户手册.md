# NeuroLab Hub 用户手册

版本：1.0.1  
适用对象：实验室管理人员、实验操作人员、部署维护人员

## 1. 手册说明

本手册描述 `1.0.1` 版本的实际交付、启动和日常使用方式。当前推荐交付形态为 `SilentDir/onedir` 轻量包，不再以 `onefile` 作为正式交付目标。

## 2. 获取与交付形态

当前推荐交付方式只有一种：

- Windows 端：`NeuroLab Hub SilentDir.exe` + `_internal/` + `APP/`
- Raspberry Pi 端：`pi/` 目录整体复制
- 分发方式：`zip` 压缩包

解压后即可使用，不依赖安装版。

## 3. Windows 端启动

### 3.1 启动步骤

1. 解压发布包。
2. 进入 `NeuroLab Hub SilentDir/` 目录。
3. 双击 `NeuroLab Hub SilentDir.exe`。
4. 等待启动自检结束后进入主界面。

注意：`_internal/` 与 `APP/` 都是运行目录，不能拆开、重命名或只保留 `exe` 单文件。

### 3.2 首次启动说明

首次启动会检查并准备以下运行时：

- Python 依赖环境
- Ollama 本地模型环境
- GPU 算力环境
- 训练运行时环境
- 实验室知识库目录

当前版本已移除“双重启动页”问题，也不再把大体量语音模型下载放在 GUI 主启动链路中。

## 4. Pi 端启动

### 4.1 推荐首轮接入方式

当前推荐由 Windows 端一键完成首轮接入：

1. 打开 `pc/pi_one_click_setup.json`
2. 至少填写：
   - `ssh.user`
   - `ssh.password`
3. 双击 `pc/一键配置树莓派.cmd`

脚本会自动：

- 发现树莓派
- 让树莓派连接到与当前 PC 相同的 Wi‑Fi
- 投递 `pi/` 目录
- 触发树莓派后台自治安装
- 安装完成后自动启动边缘节点

### 4.2 Pi 端状态命令

```bash
python3 pi_cli.py install-status
python3 pi_cli.py status
```

### 4.3 Pi 端启动命令

```bash
bash start_pi_node.sh
```

### 4.4 图形入口

`pi/` 目录内已提供：

- `start_pi_node.sh`
- `pi_cli.py`
- `NeuroLab Hub Pi.desktop`

复制到树莓派后，可通过一行命令或桌面入口启动。

## 5. GUI 主流程

推荐用户按以下顺序操作：

1. 启动主程序
2. 完成系统自检
3. 选择 AI 模型与模型名称
4. 导入公共知识
5. 导入专家资产
6. 导入专家知识
7. 创建训练工作区并导入训练数据
8. 打开标注、档案、模型配置窗口确认功能正常
9. 点击“开始监控”
10. 等待 `Pi` 节点上线并进入闭环

## 6. 当前已验证模块

`1.0.1` 已实际验证以下 GUI 模块：

- 主界面与系统自检
- 模型选择
- 公共知识导入
- 专家模型编排
- 专家知识导入
- `LLM` 训练工作区创建与数据导入
- 视觉训练工作区创建与数据导入
- 标注面板最小交互
- 档案中心刷新
- 模型配置窗口打开与最小交互
- 开始监控、多节点发现、事件流展示、状态统计

## 7. 语音与视觉闭环

### 7.1 语音闭环

当前闭环为：

1. `Pi` 本地离线识别音频为文本
2. 文本上行至 `PC`
3. `PC` 路由到知识问答、专家问答或语音调用类模型
4. `PC` 文本结果回传 `Pi`
5. `Pi` 本地播报

### 7.2 视觉闭环

1. `Pi` 上送视觉事件或关键帧
2. `PC` 调度专家模型
3. `PC` 返回结构化结果
4. `Pi` 完成 ACK 与播报

## 8. 新机首启说明

当前版本支持在清空运行时缓存后重新首启。新机模拟时，系统会重新准备用户级运行时目录，但不会要求重新打包整个程序。

## 9. 已知边界

- 真实树莓派硬件联调不在本轮文档范围内
- 当前正式交付推荐 `SilentDir/onedir`
- 树莓派首次补依赖由 Pi 后台自治安装完成，不再推荐前台手工等待
