# 实验数据集构建规范

## 1. 目标

本规范用于指导 LabDetector 在实验室真实场景下进行数据规模化采集、标注、归档和训练，确保后续大模型微调与 Pi 端识别模型微调具备可复用、可追溯的数据基础。

## 2. 数据类型

建议按以下四类组织：

- 图像数据：实验器具、实验台面、人员手部动作、风险事件截图
- 视频数据：完整实验过程、关键步骤视频、异常操作样本
- 语音数据：语音提问、专家回答、现场播报、常见问答
- 文本数据：SOP、设备说明、制度文档、实验记录、事故复盘、FAQ

## 3. 目录建议

```text
dataset_root/
├─ vision/
│  ├─ images/
│  ├─ videos/
│  ├─ labels/
│  └─ manifests/
├─ voice/
│  ├─ audio/
│  ├─ transcripts/
│  └─ manifests/
├─ text/
│  ├─ sop/
│  ├─ manuals/
│  ├─ faq/
│  └─ experiment_logs/
└─ metadata/
```

## 4. 视觉标注建议

对于 Pi 端检测模型，建议至少包含：

- `器具类型`
- `操作动作`
- `流程阶段`
- `风险等级`
- `是否规范`

YOLO 标注可用于目标检测，后续若要做动作识别，可补充时间序列标签与阶段标签。

## 5. LLM 微调样本建议

推荐使用统一的监督微调数据格式：

```json
{
  "instruction": "移液枪吸液后为什么要贴壁缓慢放液？",
  "output": "这样可以减少气泡并提升体积准确性……",
  "metadata": {
    "project_name": "微流控实验",
    "experiment_name": "液滴生成",
    "operator_name": "张三",
    "tags": ["移液", "SOP", "问答"]
  }
}
```

支持来源：

- 语音轮次归档自动生成
- 实验档案中的专家结论自动生成
- 人工整理 FAQ / SOP / 复盘问答

## 6. 采集规范

- 每个实验场景至少覆盖正常样本与异常样本
- 每个关键器具至少采集多个角度与多个光照条件
- 每个关键步骤至少包含“开始、中间、完成、异常”四类状态
- 采集时同步记录实验项目、实验人员、实验时间、实验台区域

## 7. 训练入口

当前软件训练工作台支持：

1. 导入真实 LLM 训练数据
2. 导入真实 Pi 检测数据
3. 自动合并归档样本与外部样本
4. 构建统一训练工作区
5. 启动 LLM LoRA 微调
6. 启动 Pi 检测模型微调
7. 一键顺序运行全流程

## 8. 质量控制

- LLM 数据要避免问答重复、错答和空答
- Pi 检测数据要核查标签与图像是否一一对应
- 所有样本建议附带最少元数据：项目、实验、人员、时间、来源
- 每轮训练后应保存训练参数、模型路径和评估摘要

## 9. 建议

若要进一步提高 AI for Science 价值，建议在数据集中持续补充：

- 实验现象与机理解释问答
- 失败实验原因分析样本
- 经验规则与流程优化总结
- 多学科实验器具和操作差异样本
