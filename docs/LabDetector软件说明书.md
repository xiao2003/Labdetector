# LabDetector 软件说明书

版本号：V3.0.3
软件名称：LabDetector 智能实验室监控软件
著作权人：LabDetector 软件研发组
完成日期：2026 年 3 月

## 1. 软件概述
LabDetector 是一套面向实验室场景的桌面可视化监控软件，用于统一管理实验室监控节点、运行启动自检、选择 AI 后端与模型、查看节点在线状态，并以平铺监控墙方式展示各节点画面、能力标记和实时提示信息。

软件采用桌面 EXE 交付形态，用户双击即可启动主程序，无需手动打开 Python 源码。主界面包含启动页、运行配置区、启动自检区、节点监控墙、提示日志区，以及“关于”“版权信息”“软件说明”等页面。

## 2. 主要功能
1. 启动自检：检查依赖环境、GPU、麦克风链路、离线语音模型和知识库目录等项目，输出明确状态和摘要。
2. 运行配置：提供 AI 后端、模型预设、自定义模型、运行模式和预期节点数设置。
3. 监控展示：以卡片平铺方式显示本机摄像头或多节点监控画面，并同步展示节点地址、在线状态、提示信息、麦克风/扬声器能力等信息。
4. 运行日志：集中显示启动、连接、会话切换和风险提示等日志信息。
5. 知识库管理：支持公共底座知识库和专家专属知识库的导入、查看与维护。
6. 产品信息页：提供启动页、关于页、版权页和说明文档页，便于展示软件主体、版本和著作权信息。
7. 隐藏演示模式：支持通过配置文件激活隐藏演示开关，在摄像头或树莓派会话建立后按专家顺序轮播提示，便于录制演示视频。

## 3. 运行环境
- 操作系统：Windows 10 或 Windows 11
- Python：用于构建和开发调试阶段；最终用户运行 EXE 时无需手动执行源码
- 摄像头：支持本机摄像头预览模式
- 网络：多节点监控模式下需要与边缘节点处于可通信网络环境
- 可选依赖：语音、OCR、知识库、专家分析等能力会根据本机资源自动启用或降级

## 4. 安装与启动
### 4.1 正式发布目录
正式交付以仓库根目录下的 `pc/` 与 `pi/` 为准，目录最外层默认只需要关注以下内容：
- `pc/`：PC 桌面端发布目录，入口为 `pc/LabDetector.exe`
- `pi/`：树莓派边缘端发布目录，入口为 `pi/start_pi_node.sh`

使用要求：保留完整 `pc/` 目录，不可仅复制单个 EXE 文件。

### 4.2 启动方式
- 进入 `pc/` 后直接双击 `LabDetector.exe`

软件启动后，先显示启动页，再进入主界面。

### 4.3 开发态运行
如需开发调试，可使用：

```powershell
C:\Users\Administrator\AppData\Local\Programs\Python\Python311\python.exe D:\Labdetector\launcher.py
```

兼容旧控制台模式：

```powershell
C:\Users\Administrator\AppData\Local\Programs\Python\Python311\python.exe D:\Labdetector\launcher.py --cli
```

## 5. 界面说明
### 5.1 启动页
启动页用于展示软件名称、版本、公司主体、版权信息和当前加载状态，用于软件交付和登记展示。

### 5.2 主界面
主界面由以下区域组成：
- 顶部品牌区：显示软件名称、运行说明、版本、当前状态和帮助入口
- 左侧控制区：显示运行配置、概览指标、启动自检结果
- 右侧监控墙：平铺展示各节点视频画面与节点提示信息
- 底部日志区：显示近期运行提示与系统日志

### 5.3 关于页
关于页展示软件名称、版本号、公司主体、软件用途和核心能力摘要。

### 5.4 版权页
版权页展示软件著作权声明、适用范围和第三方组件说明。

### 5.5 软件说明页
软件说明页内置本说明文档内容，并支持基础 Markdown 渲染，不再显示裸露的 `**`、`#` 等标记。

## 6. 使用流程
1. 启动软件并等待启动页完成初始化。
2. 在左侧配置区选择 AI 后端、模型预设和运行模式。
3. 根据运行模式填写预期节点数或自定义模型名。
4. 点击“运行启动自检”确认环境状态。
5. 点击“启动监控”进入监控会话。
6. 在监控墙和提示日志区查看节点画面、在线状态和提示信息。
7. 如需导入知识库，点击“知识库管理”并选择作用域后导入文件或目录。
8. 如需结束，点击“停止监控”或直接关闭软件。

树莓派边缘端可在 `pi/` 目录执行 `python3 pi_cli.py` 进入交互菜单，也可使用 `labdetector-pi status`、`labdetector-pi self-check`、`labdetector-pi start` 等命令。

## 7. 隐藏演示模式
### 7.1 激活方式
编辑 `config.ini`，加入或修改如下配置：

```ini
[shadow_demo]
enabled = True
interval_seconds = 8
```

### 7.2 生效方式
- 该开关不会在软件界面中显示。
- 修改完成后重新启动软件。
- 只有在本机摄像头会话或树莓派会话真正启动后，演示模式才会开始轮播。

### 7.3 演示效果
- 本机摄像头模式下：保持真实摄像头画面，但专家提示按顺序轮播，适合单机演示录屏。
- 树莓派模式下：当节点成功连上后，节点卡片会按专家顺序轮播提示信息，适合多节点演示视频。
- 日志区会同步输出当前轮播到的专家和对应演示说明。

## 8. 多知识库使用说明
### 8.1 知识库类型
- `common`：公共底座知识库
- `expert.<module>`：专家专属知识库

### 8.2 导入方式
- 图形界面：通过“知识库管理”窗口导入文件或目录
- 命令行：使用 `pc.knowledge_base.kb_builder`

### 8.3 推荐内容
- 公共底座知识库：实验室制度、SOP、应急流程、常见规则
- 危化品专家知识库：试剂目录、MSDS 摘要、禁忌与防护要求
- 设备专家知识库：仪器规程、校准流程、维护记录
- 问答专家知识库：术语、常见问答、实验方法说明

### 8.4 图片与视频转知识建议
- 图片：先做 OCR、标签提取、对象识别，再整理成 `txt/json/csv`
- 视频：先做 ASR、关键帧提取、事件切片，再整理成 `md/json/csv`
- 不建议将原始大视频直接作为知识库原料

## 9. 注意事项
- 多节点模式依赖实验室网络与边缘节点连通性。
- 某些 AI 模型、语音或知识库能力依赖本机环境，若资源缺失，软件会给出提示信息。
- 正式运行请只使用 `pc/LabDetector.exe`。
- 若用于软件著作权登记，建议提交启动页、主界面、关于页、版权页和说明页截图。
