# LabDetector 软件说明书

版本号：V3.0.0  
软件名称：LabDetector 智能实验室监控软件  
著作权人：LabDetector 软件研发组  
完成日期：2026 年 3 月

## 1. 软件概述
LabDetector 是一套面向实验室场景的桌面可视化软件，用于统一管理实验室监控节点、运行启动自检、选择 AI 后端与模型、查看节点在线状态，并以平铺监控墙方式展示各节点画面、能力标记和实时提示信息。

软件采用桌面 EXE 交付形态，用户双击即可启动主程序，无需手动打开 Python 源码。主界面包含启动页、运行配置区、启动自检区、节点监控墙、提示日志区，以及“关于”“版权信息”“软件说明”等页面。

## 2. 主要功能
1. 启动自检：检查版本、依赖、模型资源、语音能力、监控运行环境等项目，输出明确状态和摘要。
2. 运行配置：提供 AI 后端、模型预设、自定义模型、运行模式和预期节点数设置。
3. 监控展示：以卡片平铺方式显示本机摄像头或多节点监控画面，并同步展示节点地址、在线状态、提示信息、麦克风/扬声器能力等信息。
4. 运行日志：集中显示启动、连接、会话切换和风险提示等日志信息。
5. 知识库管理：支持公共底座知识库和专家专属知识库的导入、查看与维护。
6. 产品信息页：提供软件启动页、关于页、版权页和说明文档页，便于展示软件主体、版本和著作权信息。
7. 单入口交付：正式发布目录最外层只暴露一个 `LabDetector.exe`，依赖运行时目录会被隐藏处理。

## 3. 运行环境
- 操作系统：Windows 10 或 Windows 11
- Python：用于构建阶段，最终用户运行 EXE 时无需单独执行源码
- 摄像头：支持本机摄像头预览模式
- 网络：多节点监控模式下需要与边缘节点处于可通信网络环境
- 可选依赖：语音、OCR、知识库、专家分析等能力会根据本机资源自动启用或降级

## 4. 安装与启动
### 4.1 正式发布目录
正式交付以 `release/LabDetector-v3.0.0/` 为准，目录最外层默认只需要关注以下内容：
- `LabDetector.exe`：桌面主程序唯一对外入口
- `_internal/`：运行所需的内嵌依赖与资源，默认隐藏，不需要手动进入

使用要求：保留整个目录，不可仅复制单个 EXE 文件。

### 4.2 启动方式
- 直接双击 `LabDetector.exe`

软件启动后，先显示启动页，再进入主界面。

### 4.3 构建目录说明
- `build/` 或 PyInstaller 工作目录仅用于打包缓存，不属于交付物。
- 若看到 `build` 中的 EXE，不应直接运行。
- 正式运行请只使用 `release/LabDetector-v3.0.0/LabDetector.exe`。

## 5. 界面说明
### 5.1 启动页
启动页用于展示软件名称、版本、公司主体、版权信息和当前加载状态，用于软件交付和登记展示。

### 5.2 主界面
主界面由以下区域组成：
- 顶部品牌区：显示软件名称、运行说明、版本、当前状态和帮助入口
- 左侧控制区：显示运行配置、概览指标、启动自检结果
- 右侧监控墙：平铺展示各节点视频画面与节点提示信息
- 底部日志区：显示近期运行提示与系统日志

### 5.3 关于页
关于页展示软件名称、版本号、公司主体、软件用途和核心能力摘要。

### 5.4 版权页
版权页展示软件著作权声明、适用范围和第三方组件说明。

### 5.5 软件说明页
软件说明页内置本说明文档内容，便于软件内直接查看。

## 6. 使用流程
1. 启动软件并等待启动页完成初始化。
2. 在左侧配置区选择 AI 后端、模型预设和运行模式。
3. 根据运行模式填写预期节点数或自定义模型名。
4. 点击“运行启动自检”确认环境状态。
5. 点击“启动监控”进入监控会话。
6. 在监控墙和提示日志区查看节点画面、在线状态和提示信息。
7. 如需结束，点击“停止监控”或直接关闭软件。

- 多节点模式依赖实验室网络与边缘节点连通性。
- 某些 AI 模型、语音或知识库能力依赖本机环境，若资源缺失，软件会给出提示信息。
