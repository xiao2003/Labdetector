# NeuroLab Hub 首次运行修复手册

## 目标
本手册只覆盖首次运行和监控启动前最常见的可修复问题，修复路径与总览页仪表盘上的“查看并修复”保持一致。

## 适用范围
- 本机运行环境缺失
- 固定管家层未完成准备
- 知识库或专家目录未刷新完成
- Pi 节点未全部上线或节点链路异常

## 修复原则
- 先看总览页“总仪表盘”
- 如果出现异常卡片或修复提示，优先点击“查看并修复”
- 不需要手动改配置，不需要手动清理目录
- 修复动作完成后，系统会再次刷新状态

## 常见问题与修复方式

### 1. 本机运行环境缺失
现象：
- 会话与运行时层显示“自检中”或“受限”
- 监控无法正常启动

处理：
1. 在总览页点击“查看并修复”
2. 等待系统完成逐项补全和再次自检
3. 确认总仪表盘恢复为正常状态

说明：
- 自检进度不会再显示到关键事件日志中
- 详细补全过程仍会保留在内部运行日志

### 2. 固定管家层未完成准备
现象：
- 管家编排层显示“准备中”或“受限”
- 首次运行后仍处于规则链

处理：
1. 在总览页点击“查看并修复”
2. 系统会重新检查固定模型和运行时资源
3. 若环境允许，管家层会重新进入准备流程

说明：
- 若网络环境异常导致模型准备失败，系统会继续使用规则链，不阻塞 GUI
- 这种情况会在仪表盘中持续显示为待处理问题

### 3. 知识库或专家目录未就绪
现象：
- 执行与知识层显示“部分就绪”
- 问答、专家调度或知识增强结果不完整

处理：
1. 在总览页点击“查看并修复”
2. 系统会刷新知识库目录和专家目录
3. 等待总仪表盘状态刷新

说明：
- 该修复不会删除已有知识文件或专家资产

### 4. Pi 节点未全部上线
现象：
- 通信与节点管理层显示“待连接”或“连接中”
- 右侧“监控状态”中节点数量少于预期

处理：
1. 在总览页点击“查看并修复”
2. 系统会向已连接节点发起重新自检和状态同步
3. 再次确认节点连接、识别与告警状态

说明：
- 如果节点本身未启动，本机只能等待节点上线，无法代替节点完成启动

## 人工检查项
- 首屏只显示“系统已可用 / 后台准备中 / 后台准备失败（已回退规则链）”
- 总览页顶部操作按钮可用
- 总仪表盘 6 层状态全部可见
- 右侧“监控状态”能显示节点连接、识别、告警
- 关键事件日志只保留关键事项，不显示自检进度

## 不建议的操作
- 不要手动修改 `config.ini`
- 不要手动删除模型目录、知识库目录、专家目录
- 不要在未确认原因前重复强制重装

## 结论
首次运行出现问题时，优先使用总览页的“查看并修复”。只有当仪表盘连续多次提示同一问题且状态没有改善时，才进入人工排查。
