# NeuroLab Hub 真实树莓派接入联调清单

版本：1.0.0

## 1. 目标

本清单用于把当前已经通过虚拟闭环验证的 `NeuroLab Hub 1.0.0`，平滑接入真实树莓派并完成第一轮实机联调。

联调目标固定为以下闭环：

1. `Pi` 上电后能够启动边缘端
2. `PC` 端点击“开始监控”后能够发现并连接 `Pi`
3. `Pi` 本地语音识别后，文本可上行到 `PC`
4. `PC` 返回文本结果后，`Pi` 能本地播报
5. `Pi` 常态视觉事件可上送 `PC`
6. `PC` 专家分析结果可回传 `Pi`
7. `Pi` 对专家结果完成 ACK

## 2. 当前产品基线

当前版本已经确认的产品形态：

- Windows 端交付为轻量 `SilentDir/onedir`
- 树莓派端交付为复制 `pi` 目录即可启动
- 树莓派端首轮推荐入口：
  - Windows 一键脚本：`pc/一键配置树莓派.cmd`
  - Pi 端状态命令：`python3 pi_cli.py install-status`
  - Pi 端启动命令：`bash start_pi_node.sh`
- `PC` 端联调入口：
  - 打开 GUI
  - 点击“开始监控”

## 3. 实机联调前提

### 3.1 Windows 端

联调前必须确认：

- 已使用当前发布包解压运行，而不是源码目录随意启动
- Windows 端 GUI 已能正常进入主界面
- Ollama 已可用
- 本机防火墙未阻断局域网内 `Pi -> PC` 通信
- `PC` 与 `Pi` 在同一局域网

建议联调前先做一次：

```powershell
python -m pc.testing.virtual_text_voice_closed_loop_test
```

目的：

- 先确认当前 `PC` 端自身没有回归，再接真实硬件

### 3.2 树莓派硬件

建议最低准备：

- Raspberry Pi 4B / 5
- `64GB` 以上存储卡
- 稳定电源
- USB 麦克风或可用麦克风模组
- 可用扬声器
- 可用摄像头
- 稳定局域网

### 3.3 树莓派系统

建议基线：

- Raspberry Pi OS 64-bit
- Python 3.11 可用
- 能正常执行 `python3`
- 允许 `pip` 安装依赖
- 系统时钟正常

## 4. 树莓派部署清单

### 4.1 拷贝目录

将当前发布目录中的 [`pi`](/D:/NeuroLab/NeuroLab%20Hub/pi) 整个复制到树莓派，例如：

```bash
/home/pi/NeuroLab/pi
```

要求：

- 不要只复制单个脚本
- `pi` 目录内部结构必须完整保留
- 不要改名 `config.ini`、`voice/`、`models/`、`tools/`

### 4.2 启动前确认

进入树莓派目录后执行：

```bash
cd /home/pi/NeuroLab/pi
python3 pi_cli.py status
```

检查以下信息：

1. 配置文件路径是否正确
2. 本机 IP 是否正确
3. `PC` 中枢地址是否正确
4. `ws_port` 是否符合预期
5. 唤醒词是否正确
6. 离线模型目录解析是否正确
7. 检测权重路径是否正确

### 4.3 首轮自治安装状态

执行：

```bash
python3 pi_cli.py install-status
```

通过标准：

1. 安装状态最终为 `success`
2. 无残留失败阶段
3. 状态文件和日志路径可读

如果安装尚未完成，用户无需守在终端前等待。

### 4.4 一行启动

推荐正式启动方式：

```bash
bash start_pi_node.sh
```

无参数时默认行为：

- 自动执行 `start`
- 如果运行时尚未准备好，只返回安装状态
- 在脚本目录内启动，避免路径漂移

## 5. 树莓派配置项联调检查

联调前建议明确确认以下配置：

### 5.1 网络配置

- `network.pc_ip`
  - 明确填入 Windows 中枢地址
  - 如果不填，必须确认自动发现链路可用

- `network.ws_port`
  - 当前应与 `PC` 端监听端口一致

### 5.2 语音配置

- `voice.wake_word`
  - 与 `PC` 下发配置一致

- `voice.online_recognition`
  - 联调阶段建议先明确当前实际使用策略

- `voice.model_path`
  - 必须指向树莓派本地实际可读目录

### 5.3 视觉配置

- `detector.weights_path`
  - 必须指向真实检测权重

- `detector.conf`
  - 先维持默认值，不要联调前临时乱改

- `detector.imgsz`
  - 先维持默认值，不要联调前临时乱改

## 6. PC 端联调操作清单

真实树莓派接入时，Windows 端按以下顺序执行：

1. 启动 [`NeuroLab Hub SilentDir.exe`](/D:/NeuroLab/NeuroLab%20Hub/NeuroLab%20Hub%20SilentDir.exe)
2. 等待主界面进入
3. 在 GUI 中确认：
   - 模型选择正确
   - 专家模型已就绪
   - 知识库已导入
4. 打开训练/知识/专家窗口只做快速确认，不在首次联调中临时改大量配置
5. 切到 `websocket` 监控模式
6. 填写预期节点数
7. 点击“开始监控”

开始监控后，应重点观察：

1. 是否发现真实 `Pi`
2. 节点状态是否从离线变为在线
3. 右侧事件流日志是否出现握手和能力上报
4. 在线/离线节点数是否正确

## 7. 实机语音联调清单

### 7.1 联调目标

验证真实链路：

`Pi 用户说话 -> Pi 本地识别 -> 文本回传 PC -> PC 处理 -> 文本结果回传 Pi -> Pi 播报`

### 7.2 先测最小语音样本

建议按这个顺序测：

1. `介绍当前系统状态`
2. `分析当前实验风险并给出处置建议`
3. `HF且未戴手套怎么办`
4. `查看PPE风险`

### 7.3 通过标准

每条样本至少确认：

1. `Pi` 本地识别成功
2. `PC` 收到文本
3. `PC` 返回结果
4. `Pi` 本地播报成功

### 7.4 现场观察点

重点观察：

1. 唤醒是否稳定
2. 识别是否明显漂移
3. 是否误触发本地 GUI 控制类命令
4. 播报是否有明显延迟
5. 同一命令是否重复触发

## 8. 实机视觉联调清单

### 8.1 联调目标

验证真实链路：

`Pi 摄像头输入 -> Pi 常态检测/关键帧裁剪 -> 事件上送 PC -> PC 专家分析 -> 结果回传 Pi -> Pi ACK`

### 8.2 最小视觉场景

建议先测：

1. 危化品识别
2. PPE 穿戴检查

### 8.3 通过标准

每类事件至少确认：

1. `Pi` 侧触发事件
2. `PC` 日志出现对应事件
3. `PC` 返回专家结果
4. `Pi` 接收并 ACK
5. 需要播报时，`Pi` 正常播报

## 9. 首轮实机联调执行顺序

首次接真实树莓派，严格按这个顺序执行：

1. Windows 端确认当前发布包可正常启动
2. 树莓派端 `status`
3. 树莓派端 `self-check`
4. 树莓派端启动节点
5. Windows 端点击“开始监控”
6. 先做节点上线验证
7. 再做语音最小样本验证
8. 再做视觉最小样本验证
9. 最后做组合场景验证

不要一开始就同时测：

- 多节点
- 长时间驻留
- 高频视觉事件
- 高频语音交互

首次联调先做单节点最小闭环。

## 10. 故障定位顺序

联调失败时，按以下顺序定位，不要乱跳层：

1. 树莓派本机是否启动成功
2. 树莓派本机自检是否通过
3. `Pi -> PC` 网络是否可达
4. `PC` 是否发现节点
5. `Pi` 语音是否本地识别成功
6. `Pi` 视觉事件是否本地触发
7. `PC` 是否收到上行消息
8. `PC` 是否生成回传结果
9. `Pi` 是否收到回传并 ACK

## 11. 安全与现场约束

真实实验室联调时，必须遵守以下限制：

1. 首次联调不要在真实高危实验进行中执行
2. 危化品相关测试优先使用演示样本或静态场景
3. 不要把系统播报结果当作唯一安全依据
4. 不要在未确认模型和知识库正确前用于实际安全决策
5. 出现误报或漏报时，立即记录样本，不要现场直接改代码
