# NeuroLab Hub 软著提交口径摘要（1.0.0）

软件全称：NeuroLab Hub——可编排专家模型的实验室多模态智能中枢  
软件简称：NeuroLab Hub  
版本号：V1.0.0

## 一、提交口径总原则

本版本用于软件著作权登记时的第一版正式口径。提交时应统一采用：

1. 软件名称统一
2. 版本号统一为 `V1.0.0`
3. 文档、测试报告、说明书、发布说明、版权声明中的版本号保持一致
4. 软件形态按“可交付软件产品”描述，而不是“零散代码仓库”描述

## 二、软件形态口径

当前建议提交口径为：

- 软件类型：实验室多模态智能中枢软件
- 软件形态：PC 中心端 + Raspberry Pi 边缘端协同的软件系统
- PC 端交付：Windows 轻量启动包
- Pi 端交付：`pi/` 目录复制部署 + 自治安装脚本

这一定义既能覆盖项目实际结构，也能避免把当前版本误描述为纯原型代码集合。

## 三、建议统一对外功能描述

建议在申请表、说明书摘要、功能简介中统一使用以下口径：

NeuroLab Hub 是一套面向科研实验室场景的多模态智能中枢软件。系统采用 PC 中心端与 Raspberry Pi 边缘节点协同架构，融合计算机视觉、大语言模型、语音识别、语音播报、知识库管理、实验档案沉淀和模型训练能力，可实现实验现场监控、专家模型编排分析、知识增强问答、语音交互和训练回灌等功能。

## 四、建议统一功能模块口径

建议按以下模块归纳：

1. 桌面可视化监控模块
2. Pi 节点发现与调度模块
3. 视觉识别与专家分析模块
4. 语音智能交互模块
5. 知识库管理模块
6. 实验档案管理模块
7. LLM 微调与视觉训练模块
8. AI 后端接入模块

## 五、建议统一技术特点口径

建议统一强调：

1. 采用 PC + Raspberry Pi 的双端协同架构
2. 支持多专家模型编排与知识域增强
3. 支持实验场景的视觉识别、语义分析与结果回传
4. 支持语音输入、问答处理与播报输出
5. 支持实验档案沉淀与训练回灌
6. 支持本地模型服务与云端模型接口接入

## 六、建议统一运行环境口径

### 6.1 PC 端

- Windows 10 / Windows 11 64 位
- Python 3.11 生态运行环境
- 支持本地模型服务或 OpenAI 兼容模型服务

### 6.2 Pi 端

- Raspberry Pi 4B / 5 或同等级 ARM Linux 设备
- 摄像头、麦克风、扬声器等边缘外设按需接入
- 与 PC 中心端网络可达

## 七、建议统一测试与验证口径

当前 `1.0.0` 已建议按以下范围作为测试证明材料口径：

1. Windows 主程序启动验证
2. GUI 主链路验证
3. 知识导入、专家编排、训练入口验证
4. 单节点与四节点虚拟 Pi 闭环验证
5. 音频文件驱动的 Pi 语音闭环验证
6. 新机解压首启与换机复验验证
7. Pi 自治安装和状态链验证

应明确：
- 真实第三方音频扩展硬件兼容性，不作为本次软著版本成立的前置条件。
- 软著登记关注的是软件产品整体功能与独创表达，不要求所有外设型号都已完成最终硬件联调。

## 九、当前不建议写入“已完全通过”的内容

在软著材料中，不建议把以下内容写成“全部已通过”口径：

1. 某特定第三方音频扩展板的实机兼容性
2. 实物多节点长期稳定联调已全部完成
3. 所有外设型号都已经全部验证

建议改写为：

- 软件已形成完整交付基线与主链路验证能力；
- 边缘外设接入能力已具备工程基础；
- 特定第三方硬件仍可按设备型号继续适配。

NeuroLab Hub 当前版本应统一按 `V1.0.0` 作为第一版正式软件口径提交。该版本的软件名称、交付形态、功能描述、测试口径和版权说明已经具备统一化条件，可直接作为软著材料整理和提交的基线版本。
