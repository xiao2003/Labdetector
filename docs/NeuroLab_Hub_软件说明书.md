# NeuroLab Hub——可编排专家模型的实验室多模态智能中枢 软件说明书

软件全称：NeuroLab Hub——可编排专家模型的实验室多模态智能中枢  
软件简称：NeuroLab Hub  
版本号：1.0.0  
完成日期：2026 年 3 月  
著作权人：NeuroLab Hub 软件研发组

## 一、软件概述

NeuroLab Hub 是一套面向实验室场景的软件系统。软件以 PC 中心端与 Raspberry Pi 边缘节点协同为基础，融合视觉专家分析、知识库增强、语音交互、实验档案沉淀和训练工作台能力，为实验室安全、危化品识别、知识管理和模型持续迭代提供统一入口。

## 二、研发目标

1. 建立中心-分布式实验室智能监控体系。
2. 建立危化品与着装安全的最小可用闭环。
3. 建立语音交互、知识提取与实验归档机制。
4. 建立训练与模型回灌能力。

## 三、总体架构

### 3.1 架构组成

- PC 中心端：界面展示、专家调度、知识库检索、统一多模态解释与结果回传
- Pi 边缘端：视频采集、关键帧触发、语音采集与播报
- 专家层：危化品、着装安全等结构化视觉分析
- 知识层：`common + expert.*` 多知识库结构
- 训练层：LLM 工作台与视觉训练工作台

### 3.2 工作闭环

1. Pi 端采集画面与语音
2. Pi 按策略仅上传关键帧和事件元信息
3. PC 根据专家策略调用对应专家
4. PC 结合知识库与统一多模态模型完成专业研判
5. 结果写入事件流并回传 Pi 播报
6. 会话、关键帧和记录写入实验档案

## 五、主要功能模块

### 5.1 桌面可视化监控模块

- 主控界面
- 视频监控区
- 主系统事件流

### 5.5 知识库模块

- 公共知识库

## 六、AI 模型与运行方式

### 6.1 本地模型

本地模型优先通过 `Ollama` 使用，默认提供：

- `gemma3:4b`
- `llama3.2-vision:11b`
- `qwen3.5:4b`
- `qwen3.5:9b`
- `qwen3.5:27b`
- `qwen3.5:35b`

### 6.2 云端模型

云端模型统一通过“模型配置”配置：

- 通义千问
- DeepSeek
- Kimi

### 6.3 运行时下载策略

- 语音模型不随交付包分发，由系统自检按需下载
- Ollama 程序在自检阶段检测并安装
- Ollama 模型在首次选择时自动拉取

## 七、运行环境

### 7.1 PC 端

- Windows 10 / 11 64 位
- Python 3.11
- 建议具备独立显卡

### 7.2 Pi 端

- Raspberry Pi 4B / 5
- 摄像头、麦克风、扬声器
- 与 PC 可达的网络
