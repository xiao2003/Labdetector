## 一、软件概述

NeuroLab Hub 是一套可编排专家模型的实验室多模态智能中枢软件系统。软件以 AI for Science 为技术导向，融合计算机视觉、大语言模型、语音识别、语音播报、知识库管理、实验档案沉淀和模型训练能力，为实验室监控、风险预警、智能问答、知识传承和私有模型迭代提供统一的软件化平台。

软件采用 “PC 中心端 + Raspberry Pi 边缘节点 + 专家模型体系 + 多知识库体系 + 训练工作台” 的整体架构，可直接以 Windows 安装包、便携 ZIP 和源码仓库三种方式交付。对普通用户而言，可通过安装包或 ZIP 直接启动主程序；对科研团队和开发者而言，可继续在 `pc/`、`pi/`、`docs/` 和 `scripts/` 目录基础上扩展业务能力、知识管线和训练流程。

## 二、研发背景

实验室场景长期存在以下问题：

1. 实验过程虽有视频监控，但多数系统只能留存画面，无法理解操作语义。
2. 实验操作规范、设备手册、SOP 和经验总结分散在文档、图片和个人记忆中，知识难以积累。
3. 实验过程中的即时咨询、风险提醒和异常播报缺少统一交互入口。
4. 通用视觉模型难以直接适配实验室仪器、危化品、人员动作和专业研究场景。
5. 用户往往需要基于真实实验数据持续训练私有模型，但缺少一套与业务系统联动的一键训练流程。

NeuroLab Hub 的建设目标，是将实验室实时感知、可编排专家协同分析、语音智能交互、实验档案留痕和模型持续训练整合成一套可交付、可部署、可演示、可申报的软件系统。

## 三、建设目标

1. 建立多节点实验室视频采集、集中管理和可视化监控能力。
2. 建立实验过程的视觉识别、语义化分析和风险提示能力。
3. 建立面向实验人员的语音咨询、语音播报和文字提示能力。
4. 建立公共知识库与专家知识库并行管理机制。
5. 建立实验档案的结构化记录、检索和复盘能力。
6. 建立面向真实实验数据的 LLM 微调与识别模型训练闭环。
7. 将上述能力封装为普通用户可直接安装和运行的软件产品。

## 五、总体架构

### 5.1 架构组成

NeuroLab Hub 采用中心端与边缘端解耦的系统架构：

- PC 中心端负责节点扫描、策略下发、专家调度、知识增强、界面展示、实验档案管理和训练控制。
- Raspberry Pi 边缘节点负责摄像头接入、关键帧采集、边缘规则触发、语音采集和结果播报。
- AI 后端既支持本地模型服务，也支持云端 API。
- 训练工作台负责数据导入、工作区生成、训练执行和模型回灌。

### 5.2 工作闭环

1. Pi 节点采集实验视频流、关键帧和语音信息。
2. PC 扫描并接入多个 Pi 节点，向节点下发策略和运行参数。
3. Pi 节点在检测到事件后上报关键帧、元数据和语音内容。
4. PC 调度专家模型、知识库和大模型后端完成分析。
5. 分析结果在主界面显示，并可回传至 Pi 进行语音或文字播报。
6. 关键帧、事件、风险提示、对话轮次和处理结果同步写入实验档案。
7. 用户可将真实实验数据导入训练工作台，生成新的识别模型或 LLM 微调产物并重新投入运行。

## 六、软件组成

### 6.1 交付形态

- `NeuroLab-Hub-Setup-vX.Y.Z.exe`：Windows 引导式安装包
- `NeuroLab-Hub-vX.Y.Z.zip`：完整闭环部署包
- `NeuroLab-Hub-Portable-vX.Y.Z.zip`：Windows 便携包

### 6.2 安装后的主要入口

- `pc/NeuroLab Hub.exe`：主程序入口
- `pc/NeuroLab Hub LLM.exe`：LLM 微调工作台入口
- `pc/NeuroLab Hub Vision.exe`：识别模型训练入口
- `pi/start_pi_node.sh`：Pi 节点启动脚本

### 6.3 源码结构

- `pc/`：中心端软件、桌面界面、知识库、训练与核心逻辑
- `pi/`：边缘节点控制、摄像头采集、语音交互和通信逻辑
- `docs/`：用户文档、说明书、版权材料、测试与发布文档
- `installer/`：安装器脚本、协议文本和安装资源
- `scripts/`：打包、发布、版本和运行时准备脚本

## 七、主要功能模块

### 7.1 桌面可视化监控模块

- 提供统一的主控面板和软件品牌入口。
- 右侧监控墙平铺显示多个 Pi 节点视频回传画面。
- 支持点击单路视频进行聚焦查看。
- 顶部右侧核心功能按钮采用横向排布，便于进入知识库、专家模型、训练工作台和软件说明等页面。
- 将系统日志与监控面板融合为结构化事件流，避免跳屏刷新影响阅读。

### 7.2 节点扫描与策略下发模块

- 扫描局域网内可用 Pi 节点。
- 建立 WebSocket 或相关控制通道。
- 向节点下发监控策略、运行模式和参数配置。
- 维护多节点运行状态与心跳信息。

### 7.3 视觉检测与专家分析模块

- 接收边缘节点关键帧和元数据。
- 调用视觉专家模型进行安全检查、器具识别、场景判断和研究分析。
- 生成语义化提示、风险等级和建议操作文本。

当前已集成的主要专家包括：

- 实验室危化品识别提醒专家
- 实验室仪器操作规范专家
- 火焰烟雾风险专家
- 通用安全行为专家
- 手部姿态估计专家
- 综合实验室安全聚合专家
- 实验室装备穿戴规范专家
- 液体洒漏检测专家
- 设备 OCR 识别专家
- 实验室智能问答专家
- 微纳流体接触角分析专家
- 微纳力学多模型专家

### 7.4 语音智能交互模块

- 支持 Pi 端智能语音交互。
- 支持 PC 本机摄像头模式下的智能语音交互。
- 打通 ASR、LLM 和 TTS 流程，实现 “听取问题 -> 理解 -> 生成回答 -> 播报结果”。
- 将视觉分析结果直接转化为文字提示或语音播报内容。
- 将单轮对话按轮次归档，便于复盘和审计。

### 7.5 知识库管理模块

- 支持公共底座知识库 `common`。
- 支持面向不同专家模型的 `expert.*` 专属知识库。
- 支持导入文本、Markdown、图片、音频和视频资料。
- 支持结构化存储与向量检索并行组织。
- 可为实验 SOP、设备手册、FAQ、历史记录和研究资料提供统一知识底座。

### 7.6 实验档案管理模块

- 按会话和时间线记录实验过程。
- 归档关键帧、事件、对话轮次和风险提示。
- 支撑实验追溯、示范视频制作、知识沉淀和管理复盘。

### 7.7 训练工作台模块

- 支持导入真实实验数据。
- 支持构建 LLM 微调工作区。
- 支持构建识别模型训练工作区。
- 支持训练依赖检测与运行时安装。
- 支持训练日志记录、训练结果回灌和模型注册。

### 7.8 AI 后端接入模块

软件通过统一 AI 后端接口兼容多类模型服务，当前支持：

- Ollama
- OpenAI 兼容服务
- LM Studio
- vLLM
- SGLang
- LMDeploy
- Xinference
- llama.cpp
- OpenAI、DeepSeek 等云端 API

上述设计便于用户按算力、网络、合规和成本要求选择本地或云端模型服务。

## 八、运行环境

### 8.1 PC 端推荐环境

- 操作系统：Windows 10 / Windows 11 64 位
- CPU：8 核以上推荐
- 内存：32GB 及以上推荐
- GPU：建议具备 NVIDIA RTX 系列独立显卡
- 磁盘：建议预留 50GB 以上空间
- 网络：与 Pi 节点位于同一局域网，或具备稳定可达链路

### 8.2 Pi 端推荐环境

- Raspberry Pi 4B / 5 或同等级 ARM Linux 设备
- 摄像头模块或 USB 摄像头
- 麦克风与扬声器
- 稳定网络连接
- Python 运行环境和基础多媒体依赖

### 8.3 模型与训练环境

- 本地模型服务可部署在 PC 上
- 若需要 LLM 微调或识别模型训练，建议配备较高显存的 GPU
- 对普通用户，训练入口已提供运行时和依赖自检流程

## 九、用户操作流程

### 9.1 普通用户基础流程

1. 安装或解压软件。
2. 启动 `NeuroLab Hub.exe`。
3. 执行软件自检，确认依赖和运行状态。
4. 选择 AI 后端并填写本地或云端模型服务参数。
5. 导入公共知识库和专家知识库资料。
6. 扫描并接入 Pi 节点。
7. 启动监控，观察监控墙和系统事件流。

### 9.2 含 Pi 的完整闭环流程

1. 在 PC 端填写 `pc/pi_one_click_setup.json` 中的树莓派用户名与密码。
2. 在 PC 端双击 `pc/一键配置树莓派.cmd`，完成首轮接入、Wi‑Fi 配置、代码投递和后台自治安装。
3. 在 Pi 端查看 `python3 pi_cli.py install-status`，确认安装完成。
4. 在 Pi 端执行 `bash start_pi_node.sh`。
5. 在 PC 端扫描并接入 Pi。
6. 下发策略。
7. Pi 回传关键帧或语音文本。
8. PC 完成专家分析与知识增强。
9. PC 将结果回传 Pi，由 Pi 进行语音或文字播报。

### 9.3 模型训练流程

1. 打开 `NeuroLab Hub LLM.exe` 或 `NeuroLab Hub Vision.exe`。
2. 导入真实实验数据。
3. 构建训练工作区。
4. 检查训练依赖和运行时。
5. 启动训练任务。
6. 查看训练日志和产物。
7. 将产物注册到运行环境并重新投入使用。

1. 软件形态完整，可直接以安装包或便携包交付。
2. 支持 PC-Pi 双端协同，具备多节点视频接入和回传能力。
3. 支持视觉分析、知识增强、语音交互和档案沉淀的多模块协同。

## 十二、软件安全与运维说明

1. 软件支持本地模型优先部署，适合对实验数据隐私要求较高的场景。
2. AI 后端、模型服务和 API Key 由用户自行配置。
3. PC 与 Pi 端均提供自检机制，便于发现依赖缺失或配置异常。
4. 推荐定期导出知识库、档案和训练产物。
5. 推荐在演示环境和生产环境分别维护独立配置与数据目录。
