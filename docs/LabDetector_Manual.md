# NeuroLab Hub 用户手册

版本：3.0.4

## 1. 软件简介

NeuroLab Hub 是一套面向实验室场景的软件化平台，覆盖 PC 端可视化、Pi 端节点接入、语音交互、知识库管理、实验档案沉淀和模型训练工作台，适用于实验室监控、风险提示、智能咨询和实验知识积累等场景。

## 2. 适用对象

- 实验室管理人员
- 实验操作人员
- 负责 AI 模型维护的技术人员
- 负责多节点部署的设备管理员

## 3. 软件组成

- Windows 主程序：`pc/Lab.exe`、`pc/LabDetector.exe`
- 控制面板：`pc/LabPanel.exe`、`pc/LabDetectorPanel.exe`
- 训练工作台：`pc/LabTraining.exe`、`pc/LabDetectorTraining.exe`
- 树莓派入口：`pi/start_pi_node.sh`

## 4. 主界面说明

### 4.1 顶部品牌区

- 显示软件名称、版本号和当前运行状态。
- 右上角核心操作按钮横向排列，可直接进入自检、专家模型、知识库、模型服务、训练工作台、软件说明和版权页面。

### 4.2 左侧配置区

- AI 后端选择
- 模型预设选择
- 自定义模型输入
- 运行模式选择
- 预期节点数设置
- 启动监控与停止监控
- 系统概览与启动自检结果展示

### 4.3 右侧主工作区

- 上半部分为多节点监控墙，平铺展示多个 Pi 节点视频。
- 下半部分为系统事件流，以结构化表格方式展示时间、级别、模块和摘要。
- 事件流下方展示当前选中事件的详细内容，方便排查。

## 5. 系统事件流

系统事件流替代了原先频繁跳屏的纯文本日志区，特点如下：

- 采用结构化表格展示
- 自动识别启动自检、模型服务、知识库、训练、语音交互、节点通信等模块
- 仅在用户位于底部查看时自动跟随最新记录
- 支持点击任意行查看详细事件内容

## 6. 专家模型管理

软件支持通过界面导入专家模型，并以中文名称显示，不直接暴露 `xxx.py` 文件名。用户可在该界面中完成：

- 查看已注册专家
- 导入新专家模型资产
- 绑定或调整对应知识库
- 查看专家状态和分类

## 7. 知识库管理

### 7.1 知识库类型

- `common`：公共背景知识库
- `expert.*`：专家专属知识库

### 7.2 导入资料类型

- 文本文档
- Markdown
- 图片
- 音频
- 视频

### 7.3 导入效果

- 原始资料归档
- 元数据记录
- 结构化摘要生成
- 纳入检索与问答链路

## 8. 运行模式

### 8.1 本机摄像头模式

- 由 PC 本机完成视频、语音和模型调用。
- 适合单机演示和工作站部署。

### 8.2 WebSocket 多节点模式

- 由多个 Pi 节点提供视频和关键帧。
- 由 PC 统一调度专家模型并回传播报结果。

## 9. 启动自检

启动自检用于保障普通用户可直接使用，主要检查：

- Python 运行时
- 必需依赖和训练依赖
- 关键目录与资源文件
- 模型服务连接可用性
- 训练工作台运行条件

当发现缺失依赖时，系统会在允许的情况下执行自动安装并复检。

## 10. 训练工作台

### 10.1 LLM 微调

适用于实验问答、SOP 提示、专业术语适配和实验记录表达风格统一。

### 10.2 YOLO 训练

适用于实验器具检测、区域识别、危险动作识别和实验步骤检测。

### 10.3 训练流程

1. 导入训练数据
2. 构建训练工作区
3. 检查训练依赖
4. 一键开始训练
5. 自动注册训练产物
6. 回到生产监控界面投入使用

## 11. 实验档案中心

实验档案围绕实验项目、实验名称、操作人员和时间进行组织，内容包括：

- 事件日志
- 关键帧
- 语音轮次
- 风险告警
- 分析结论

## 12. Pi 端使用方式

### 12.1 自检

```bash
./start_pi_node.sh self-check --auto-install-deps
```

### 12.2 启动

```bash
./start_pi_node.sh start --auto-install-deps
```

## 13. 常见问题

### 13.1 为什么文件名仍包含 LabDetector

当前发布文件名保留历史命名以兼容已有发布流程，但安装后的展示名称和主界面名称统一为 `NeuroLab Hub`。

### 13.2 为什么日志不再大幅滚动

系统事件流已改为结构化表格，仅在用户位于底部查看时才自动跟随最新记录。
